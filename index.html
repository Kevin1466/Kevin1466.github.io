Day02-机器学习基本概念2

Posted on 2022-01-16

一、为什么一些场景中使用余弦相似度而不是欧式距离？

  • 余弦距离

    • 适用场景：余弦相似度衡量的是维度间取值方向的一致性，注重维度之间的差异，不注重数值上的差异。
    • 举例：商品A从100块降价到50块，商品B从10000块降价到5000块，这两种商品价格的变化趋势相同，但是从商品价格本身来说，两者降价的绝对量差距很大。

  • 欧式距离

    • 是数值上的差异性

    • 举例：对产品用户做聚类，区分高价值用户和低价值用户，用消费次数和平均消费额来衡量，这时用余弦夹角是不恰当的，因为(2,10)和(10,50)会被认为是相似用户，但显然后者价值高很多。这时候需要注重数值上的差异，而不是维度之间的差异。

二、 在模型评估过程中，过拟合和欠拟合具体指什么现象

模型对训练集数据的误差称为经验误差，对测试集数据的误差称为泛化误差；模型对训练集以外样本的预测能力称为模型的泛化能力，追求泛化能力是机器学习的目标。

过拟合和欠拟合是导致模型泛化能力不高的两种常见原因，都是模型学习能力与数据复杂度之间失配的结果。“欠拟合”是指模型学习能力较弱，而数据复杂度较高，此时由于模型学习能力不足，无法学习到数据集中的“一般规律”，而导致泛化能力弱；相反，“过拟合”是指学习能力过强，以至于将训练集单个样本自身的特点都能捕捉到，并将其认为是“一般规律”，这种情况也会导致模型泛化能力不足。

过拟合和欠拟合的区别在于，欠拟合在训练集和测试集上的性能都较差，而过拟合往往能很好地学习训练集数据的性质，而在测试集上的性能较差。在神经网络训练的过程中，欠拟合表现为输出结果的高偏差，而过拟合则表现为结果的高方差。

三、 降低过拟合和欠拟合的方法

  1. 欠拟合的原因
    • 模型复杂度过低
    • 特征量过少
  2. 欠拟合的解决方法
    • 增加新特征，可以考虑加入特征组合、高次特征来增大假设空间；
    • 添加多项式特征，例如将线性模型通过添加二次项、三次项使模型泛化能力更强；
    • 减少正则化参数，正则化的目的是用来防止过拟合的，但既然出现了欠拟合，就应该减少正则化参数；
    • 使用非线性模型，如核SVM、决策树、深度学习模型；
    • 调整模型的容量（指拟合各种函数的能力，容量低的模型可能很难拟合训练集，使用集成学习方法，如Bagging，将多个弱学习器集成起来）；
  3. 过拟合的原因
    • 建模样本选择不当，如样本数量太少，采样方法错误，样本标签错误等，导致选取的样本数据不足以代表预定的分类规则；
    • 样本噪音干扰过大，使得机器将部分噪音认为是特征从而扰乱了预设的分类规则；
    • 参数太多，模型复杂度过高；
    • 对于决策树模型，如果对于其生长没有合理的限制，其自由生长有可能使节点只包含单纯的事件数据或非事件数据，使其虽然可以完美匹配（拟合）训练数据，但是无法适应其他数据集；
    • 对于神经网络模型，a) 对样本数据可能存在分类决策面不唯一，随着学习的进行，BP算法使权值可能收敛于过于复杂的决策面；b) 权值学习迭代次数足够多，拟合了训练数据中的噪声和训练样例中没有代表性的特征；
    • 假设的模型无法合理存在，或者说假设成立的条件实际并不成立；
  4. 过拟合的解决方法
    • 正则化（L1, L2）；
    • 数据扩增，即增加训练数据样本；
    • Dropout
    • Early Stopping

四、L1和L2正则先验分别服从什么分布

正则化是一种回归的形式，它将系数估计朝零的方向进行约束、调整或缩小。也就是说，正则化可以在学习过程中降低模型复杂度和不稳定程度，从而避免过拟合的危险。

  • 正则化解决过拟合问题

    正则化通过降低模型的复杂性，达到避免过拟合的问题。两种解释：

    1. 模型过于复杂是因为模型尝试去兼顾各个测试数据点，导致模型函数处于动荡的状态，这意味着函数在某些小区间的导数值非常大，由于自变量值可大可小，所以只有系数足够大，才能保证导数值很大。而加入正则化能抑制系数过大的问题。

      如果发生过拟合，参数θ一般是比较大的值，加入惩罚项后，只要控制λ的大小，当λ很大时，θ就会很小，即达到了约束数量庞大的特征的目的。

    2. 从贝叶斯的角度，正则化是为模型参数估计增加一个先验知识，先验知识会引导损失函数最小值过程朝着约束方向迭代。L1是拉普拉斯先验，L2是高斯先验。整个最优化问题可以看做是一个最大后验估计，其中正则化项对应后验估计中的先验，贝叶斯方法通过最大化后验概率估计参数θ。

      p(θ) 是参数向量θ的先验概率。

  • L1, L2的比较

    L1比L2更易于得到稀疏解。

    岭回归和Lasso的解都要在平方误差项与正则化项之间折中。岭回归一个明显的缺点：模型的可解释性。它将把不重要的预测因子的系数缩小到趋近于 0，但永不达到 0。也就是说，最终的模型会包含所有的预测因子。但是，在 Lasso 中，如果将调整因子 λ 调整得足够大，L1 范数惩罚可以迫使一些系数估计值完全等于 0。因此，Lasso 可以进行变量选择，产生稀疏模型。

    w取得稀疏解意味着初始的d个特征中仅有对应着w的非零分量的特征才会出现在最终模型中，于是求解L1的结果得到了仅采用一部分初始特征的模型；换言之，基于L1正则化的学习方法就是一种嵌入式特征选择方法，其特征选择过程和学习器训练过程融为一体，同时完成。

​ 总结：

    1. L1 regularizer ：它的优良性质是能产生稀疏性，导致 W 中许多项变成零。 稀疏的解除了计算量上的好处之外，更重要的是更具有“可解释性”。
      2. L2 regularizer ：使得模型的解偏向于范数较小的 W，通过限制 W 范数的大小实现了对模型空间的限制，从而在一定程度上避免了 overfitting 。不过 ridge regression 并不具有产生稀疏解的能力，得到的系数仍然需要数据中的所有特征才能计算预测结果，从计算量上来说并没有得到改观。

五、树形结构为什么不需要归一化？

机器学习中需要归一化的算法有SVM，逻辑回归，神经网络，KNN，线性回归，而树形结构不需要归一化，因为它不关心变量的值，二是关心变量分布和变量之间的条件概率，如决策树、随机森林。对于树形结构，树模型的构造是通过寻找最优分裂点构成的，样本点的数值缩放不影响分裂点的位置，对树模型的结构不造成影响，而且树模型不能进行梯度下降，因为树模型是阶跃的，阶跃是不可导的，而且求导没意义，也不需要归一化。

概率模型不需要归一化，因为它们不关心变量的值，而是关心变量的分布和比变量之间的条件概率，如决策树，随机森林。而像Adaboost，SVM，LR，KNN，KMeans之类的最优化问题就需要归一化。

对于需要归一化的模型，主要是因为特征值相差很大时，运用梯度下降，损失等高线是椭圆形，需要进行多次迭代才能到达最优点；如果进行归一化了，那么等高线就是圆形，促使SGD往原点迭代，从而减少需要迭代的次数。

Day01-机器学习一些基本概念

一、简述机器学习项目的一般流程

  1. 定义问题并看整体

    • 用业务术语定义目标
    • 当前有什么解决方案/解决方法
    • 如何阐述这个问题（有监督？无监督，在线？离线等）
    • 应该如何衡量性能
    • 性能指标是否符合业务目标
    • 达到业务目标所需的最低性能是多少
    • 有没有一些相似的问题？可以重用经验或工具吗
    • 列出所做的假设
    • 验证假设

2. 获取数据

  • 列出所需的数据以及需要多少数据
  • 检查将占用多少空间
  • 检查法律义务，并在必要时获得授权
  • 获取访问授权
  • 创建一个工作空间
  • 获取数据
  • 将数据转换为可以轻松操作的格式
  • 确保敏感信息被删除或受保护
  • 检查数据的大小和类型（时间序列、样本、地理等）
  • 抽样一个测试集，放置一遍

  1. 研究数据

    1. 创建数据副本来进行研究（必要时将其采样到可以管理的大小）

    2. 创建Jupyter notebook以记录你的数据研究

    3. 研究每个属性及其特征

      • 名称

      • 类型（分类、整数/浮点型、有界/无界、文本、结构化）

      • 缺失值的百分比

      • 噪声和噪声类型（随机、异常值、舍入误差等）

      • 分布类型（高斯分布、均匀分布、对数分布）

    4. 对于有监督的学习任务，确定目标属性

    5. 可视化数据

    6. 研究属性之间的相关性

  2. 准备数据

    1. 数据清理

      • 修复或删除异常值
      • 填写缺失值（如零、均值、中位数）或删除其行或列

    2. 特征选择

      • 删除没有为任务提供有用信息的属性
    3. 特征工程
      • 离散化连续特征
      • 分解特征
      • 添加有希望的特征转换
      • 将特征聚合为有希望的新特征
    4. 特征缩放
      • 标准化或归一化

  3. 列出有前途的模型

    1. 使用标准参数训练
    2. 衡量并比较性能

    • 对于每个模型，使用N折交叉验证，在N折上计算性能度量的均值和标准差
      3. 分析每种算法最重要的变量
      4. 快速进行特征选择和特征工程
      5. 筛选出前三到五个最有希望的模型，优先选择会产生不同类型错误的模型

  4. 微调系统

    1. 使用交叉验证微调超参数
    2. 尝试使用集成方法。组合最好的模型通常会比单独运行有更好的性能
    3. 一旦对模最终模型有信心，就可以在测试集中测量其性能，以估计泛化误差

  5. 启动

    1. 使你的解决方案准备投入生产环境（插入生产数据输入、编写单元测试）
    2. 编写监控代码，以定期检查系统的实时性能，并在系统故障时触发警报

    • 当心缓慢的退化：随着数据的发展，模型往往会”腐烂“
    • 评估性能可能需要人工流水线（例如众包服务）
    • 监视你的输入的质量，这对于在线学习系统尤其重要
      3. 定期根据新数据重新训练模型（自动化）哪些机器学习算法需要做特征归一化，哪些不需要？为什么？

二、 什么是特征归一化

好的特征不仅能够表示出数据的主要特点，还应该符合模型的假设，因此通常必须进行数据转换。数据转换有：对数变换（指数变化的一种特殊形式），特征缩放/归一化/标准化；

术语辨析：特征缩放，特征归一化，特征标准化

  • 特征缩放是使所有特征都在相似的范围内，因此建模时每个特征都变得同等重要，这样便于机器学习算法进行处理；很多机器学习模型（如k-means，knn）都依据欧氏距离，特征缩放可以将全部的数值统一到一个量级上来解决问题，为了达到这个目标，有两种基本的解决方法——标准化和归一化；

  • 标准化（Standardization）：中心标准化

    这种将特征值域缩放到0~1之间的技巧对于优化算法很有用，例如在回归和神经网络中应用到的“梯度下降”，缩放也适用于基于距离测量的算法，比如KNN；

    为了便于记忆，可以结合其英文名记作”标准化“；

  • 归一化（Max-Min Normalization）：离差标准化

    为了便于记忆，可以结合其英文名记作”正规化“

  • 对比。如果特征列中含有异常值（outlier），正规化只能将所有特征统一比例，并不能很好地解决异常值问题，而标准化在异常值方面有更好的表现，因此应用范围更广；

  1. 为什么要做归一化

    • 因为基于距离矩阵的机器学习模型（被称为“距离分类器”），如KNN, SVM, ANN，特征缩放对于这些模型非常重要，特征缩放减少了大值域变量对计算距离的影响；
    • 正规化帮助我们转化不同比例上的数据，消除特殊特征的主导，但是不能很好地处理异常值；相反，标准化可以更好地处理异常值，以及加速梯度下降算法的收敛；

三、One-hot的作用是什么？为什么不直接使用数字作为表示

One-hot是一种分类变量的编码方法，作用就是将非数值型的类别转换为数值。通常类别编码方法有

  • one-hot encoding

    每个比特位表示一个特征，因此一个可能有k个类别的分类变量就可以编码为一个长度为k的特征向量。

  • dummy coding

    没有被使用的那个特征通过一个全0向量来表示，它称为参照类。

  • effect coding

    和dummy coding的区别在于参照类是用全部由-1组成的向量表示。

  • 对比

    one-hot编码有冗余，这会使得同一个问题有多个有效模型，这种非唯一性有时候难以解释；后两种没有冗余，而且可以生成唯一的可解释的模型，dummy coding不太容易处理缺失数据，因为全0向量已经映射为参照类了；effect coding用全-1表示参照类，所以计算和存储的成本比较高；

​ 也是可以直接用整数来表示分类类别的。

四、什么是数据不平衡，如何解决？

  • 数据不平衡就是属于某一类别的观测样本的数量显著少于其他类别。当面临不平衡数据集的时候，机器学习算法倾向于产生不太令人满意的分类器。

  • 分类算法表现的评估是用一个包含关于实际类别和预测类别信息的混淆矩阵来衡量的。在不平衡领域，准确率不是一个用来衡量模型性能的合适指标。

  • 从数据角度——重采样技术

    • 随机欠采样 Random Under-Samping

      目标是通过随机地消除占多数的类的样本来平衡类的分布，直到多数类和少数类的实例实现平衡。

      缺点：会丢弃对构建规则分类器很重要的有价值的潜在信息；被随机欠采样选取的样本可能具有偏差。

    • 随机过采样 Random Over-Samping

      通过随机复制少数类来增加其中的实例数量，从而可增加样本中少数类的代表性。

      缺点：由于复制少数类事件，它加大了过拟合的可能。

    • 基于聚类过采样 Cluster-Based Over Sampling

    • 信息性过采样：合成少数类过采样技术 SMOTE

  • 从模型角度——算法集成技术

    • 基于Bagging（Bootstrap Aggregating）的方法；缺点：只会在基本分类器效果很好时才有效，错误的分类可能会进一步降低表现。
    • 基于Boosting的方法。开始于在训练数据上准备的基本分类器/弱分类器。
      • Ada Boost
      • Gradient Boosting
      • XGBoost（Extreme Gradient Boosting）
